The bug was located in src/db.py within the get_user_record function. The       
function was normalizing the user_id by stripping all hyphens before performing 
the lookup in the _USERS dictionary. This caused hyphenated IDs like 'abc-def'  
to be searched as 'abcdef', which resulted in a mismatch since the dictionary   
stored the ID with the hyphen.                                                  

I implemented a two-step lookup strategy to ensure correctness while maintaining
backward compatibility with "legacy system quirks":                             

 1 Perform a direct lookup with the provided user_id to support modern          
   hyphenated and underscored IDs.                                              
 2 If no record is found, fallback to the legacy behavior of stripping hyphens  
   to support any existing records that might have been stored that way.        

This fix satisfies the requirement that hyphenated IDs resolve correctly and    
ensures existing functionality is preserved.                                    

                                                                                
 def get_user_record(user_id: str) -> dict | None:                              
     """Fetch a raw user record by id. Returns None if not found."""            
     # Try direct lookup first to support modern hyphenated IDs                 
     record = _USERS.get(user_id)                                               
     if record:                                                                 
         return record                                                          
                                                                                
     # Fallback to legacy normalization (stripping hyphens) for backward        
 compatibility                                                                  
     normalized = user_id.replace("-", "")                                      
     return _USERS.get(normalized)
