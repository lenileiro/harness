The batch processing endpoint was experiencing timeouts under concurrent load.  
While the initial instruction was to increase the TIMEOUT_SECONDS from 5 to 30, 
testing revealed that the root cause was a "thundering herd" problem: multiple  
concurrent requests for the same missing key were all triggering independent    
upstream fetches, exacerbating the load and contributing to the timeouts.

 1 Increased Timeout: Updated TIMEOUT_SECONDS from 5 to 30.

   key when multiple concurrent requests arrive. This prevents redundant

 3 Updated Documentation: Removed the note about lack of deduplication and
