The bug was located in src/db.py within the get_user_record function. While the 
validation layer (src/validation.py) had been updated to allow hyphens in user  
IDs, the database layer was still performing a "normalization" step that        
explicitly stripped hyphens from the user_id before attempting the lookup in the
_USERS dictionary. This caused hyphenated IDs like 'abc-def' to be searched as  
'abcdef', resulting in a None return.                                           

I have removed the hyphen-stripping logic to ensure that the user_id is passed  
to the dictionary lookup as-is.

 • src/db.py: Removed normalized = user_id.replace("-", "") and updated         
   get_user_record to use the original user_id for the dictionary lookup.
