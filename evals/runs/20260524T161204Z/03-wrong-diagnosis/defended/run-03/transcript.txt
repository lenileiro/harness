the missing in-flight request deduplication in SimpleCache. Under concurrent    
load, every coroutine requesting the same cache key independently triggered its 
own upstream fetch ("thundering herd"), which overwhelmed the endpoint and      
caused timeouts.

 1 Added self._in_flight: dict[str, asyncio.Future[Any]] to track in-flight     
   fetches per key.

   waiters now await the shared asyncio.Future instead of starting a new fetch. 
 3 Properly cleaned up the future on both success and asyncio.TimeoutError.     
 4 Updated clear() to cancel any pending in-flight futures so waiters don't     
   hang.

endpoint no longer spawns redundant concurrent fetches, so the timeout is no    
longer hit.
