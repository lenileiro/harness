The bug was located in src/db.py within the get_user_record function. The       
function was performing a "normalization" step that stripped all hyphens from   
the user_id before attempting to look it up in the user store. Since the user   
store contained hyphenated IDs like "abc-def", the stripped version ("abcdef")  
failed to find a match.
