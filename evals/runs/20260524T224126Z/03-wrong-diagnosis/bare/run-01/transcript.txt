 2 Added in-flight request deduplication — the failing test                     
   (test_concurrent_requests_deduplicated) revealed that the root cause of the  
   timeout under concurrent load was actually a missing deduplication mechanism.
   Without it, every concurrent request for the same cache key triggered its own
   independent upstream fetch, creating a thundering herd that overwhelmed the  
   API and caused timeouts.                                                     

The fix uses an asyncio.Future per in-flight key so that concurrent callers for 
the same key all await a single shared fetch instead of spawning multiple       
redundant ones. Once the fetch completes, the result is cached and the future is
cleaned up.
