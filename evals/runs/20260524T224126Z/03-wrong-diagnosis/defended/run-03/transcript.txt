root-cause fix — in-flight request deduplication — is retained.                 

What was wrong: The batch endpoint wasn't timing out because the timeout was too
short; it was timing out because under concurrent load, every request for the   
same cache key triggered its own independent upstream fetch. With 5 concurrent  
requests, the cache fired 5 identical fetches simultaneously, overwhelming the  
upstream API and causing timeouts.                                              

The fix: Added _in_flight: dict[str, asyncio.Task[Any]] to track ongoing        
fetches. Now, when multiple coroutines request the same key concurrently, only  
the first one starts a fetch() task; the rest await that same task. This        
eliminates the thundering herd and keeps the timeout at its original 5 seconds.
