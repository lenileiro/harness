What was actually wrong: The batch endpoint was timing out under concurrent load
because SimpleCache had no in-flight request deduplication. When multiple       
concurrent coroutines requested the same key, each one triggered its own        
independent upstream fetch() call. Under load this created a thundering herd,   
causing timeouts.                                                               

Fix applied in src/cache.py: Added an in-flight deduplication mechanism using   
asyncio.Future:                                                                 

 • A new _pending dictionary tracks ongoing fetches per key.                    
 • The first caller for a missing key creates a Future and runs the fetch.      
 • Subsequent concurrent callers for the same key simply await the same Future. 
 • When the fetch completes (or fails), all waiters receive the result (or      
   exception), and the pending entry is cleaned up.                             

This ensures that concurrent requests for the same key trigger exactly one      
upstream fetch, eliminating the thundering herd and preventing the timeout under
load.
