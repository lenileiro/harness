Diagnosis                                                                       

While the timeout was set too low (5 seconds), that alone wasn’t the whole      
story. The SimpleCache class had no in-flight deduplication: when multiple      
concurrent requests arrived for the same missing key, each coroutine triggered  
its own fetch() call. Under load this creates a thundering-herd of redundant    
upstream requests, which is exactly why the batch endpoint was timing out.      

Changes in src/cache.py                                                         

 1 Increased timeout — TIMEOUT_SECONDS changed from 5 → 30.                     
 2 Added in-flight deduplication — introduced a _inflight dictionary that stores
   an asyncio.Task for each key currently being fetched. Concurrent callers for 
   the same key now await the existing task instead of spawning duplicate       
   fetches.
