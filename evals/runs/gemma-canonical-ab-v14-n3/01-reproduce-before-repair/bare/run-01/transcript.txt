The bug was located in src/db.py, where the get_user_record function was        
incorrectly "normalizing" the user_id by stripping all hyphens before performing
the dictionary lookup. This caused hyphenated IDs like 'abc-def' to be searched 
as 'abcdef', resulting in a mismatch with the stored keys.                      

I have removed the incorrect normalization step so that the user_id is used     
as-is for the lookup, which correctly resolves hyphenated IDs.

     # BUG: this strips hyphens, so "abc-def" becomes "abcdef" — no match.      
     normalized = user_id.replace("-", "")                                      
     return _USERS.get(normalized)
