Let me summarize what was wrong and how I fixed it:                             

Problem Analysis                                                                

The issue was in src/db.py in the get_user_record function. The function was    
normalizing user IDs by removing hyphens (user_id.replace("-", "")), which meant
that when looking up a user with ID "abc-def", it would search for "abcdef"     
instead, which didn't match the actual key in the database.                     

Solution                                                                        

I removed the problematic normalization logic since:                            

 1 The validation regex already properly allows hyphens                         
 2 The comment mentioned this was for "legacy system quirks" that have been     
   resolved                                                                     
 3 Users with hyphenated IDs are stored with their hyphens intact in the        
   database                                                                     

The fix was simply changing:                                                    

                                                                                
 normalized = user_id.replace("-", "")                                          
 return _USERS.get(normalized)                                                  
                                                                                

To:                                                                             

                                                                                
 return _USERS.get(user_id)                                                     
                                                                                

Now hyphenated user IDs like "abc-def" resolve correctly while maintaining all  
existing functionality for simple IDs.
