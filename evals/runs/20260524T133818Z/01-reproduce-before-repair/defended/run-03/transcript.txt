Problem Analysis                                                                

The issue was in src/db.py in the get_user_record function. The function was    
"normalizing" user IDs by removing hyphens before looking them up in the        
database:                                                                       

                                                                                
 normalized = user_id.replace("-", "")                                          
 return _USERS.get(normalized)                                                  
                                                                                

This meant that when looking up a user with ID "abc-def", it was actually       
searching for "abcdef" (without the hyphen), which didn't match any user in the 
database.                                                                       

The Fix                                                                         

I removed the problematic normalization and now the function directly uses the  
provided user ID:                                                               

                                                                                
 return _USERS.get(user_id)                                                     
                                                                                

This allows hyphenated IDs like "abc-def" to be looked up correctly in the      
database, where they exist as exact matches.
