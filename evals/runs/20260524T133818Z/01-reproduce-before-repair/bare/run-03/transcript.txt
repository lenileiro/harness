The bug has been successfully fixed. Here's a summary of what was done:         

Problem Analysis                                                                

The issue was that get_user() returned None for hyphenated IDs like 'abc-def'   
while simple IDs like 'alice' worked fine.                                      

Root Cause                                                                      

The bug was in src/db.py in the get_user_record function. It was normalizing    
user IDs by removing hyphens:                                                   

                                                                                
 normalized = user_id.replace("-", "")                                          
                                                                                

This caused "abc-def" to become "abcdef", which didn't match the actual stored  
key "abc-def" in the database.                                                  

Fix Applied                                                                     

I removed the hyphen normalization logic from get_user_record, allowing user IDs
with hyphens to be looked up correctly:                                         

                                                                                
 def get_user_record(user_id: str) -> dict | None:                              
     """Fetch a raw user record by id. Returns None if not found."""            
     # Fetch user record directly using the provided ID                         
     return _USERS.get(user_id)

 • Invalid IDs still raise appropriate validation errors

ensuring consistent behavior throughout the user lookup process.
